08

扩展体系：MCP、Skills、Plugins

能力扩展分成三种表面：MCP 负责外部系统接入，Skills 负责流程复用，Plugins 负责深层运行时扩展。

先分层：三种扩展机制解决三类问题

🌐

MCP

通过协议把外部工具/资源接入代理，重点是连接与隔离边界。

🧩

Skills

把重复工作流沉淀成可复用命令，重点是效率和一致性。

🪝

Plugins

在运行时关键节点注入能力，重点是深度集成与治理。

🧭
默认优先低耦合层

能用 Skills 解决就别先写 Plugin；能用 MCP 解决外部接入，就别把协议逻辑塞进 prompt。

MCP 命名空间保证能力身份不冲突

`source/claude-code-source/src/services/mcp/mcpStringUtils.ts`（第 35-52 行）

CODE 
export function getMcpPrefix(serverName: string): string {
  return `mcp__${normalizeNameForMCP(serverName)}__`
}

export function buildMcpToolName(serverName: string, toolName: string): string {
  return `${getMcpPrefix(serverName)}${normalizeNameForMCP(toolName)}`
}
中文解读

MCP 工具名采用 `mcp__server__tool` 规范，不和内置工具抢名字。

权限匹配因此可以精确到“哪个服务器上的哪个工具”。

支持资源能力的 MCP 连接会自动注入读资源工具

`source/claude-code-source/src/services/mcp/client.ts`（第 2171-2193 行）

CODE 
const supportsResources = !!client.capabilities?.resources
const resourceTools: Tool[] = []
if (supportsResources) {
  const hasResourceTools = [ListMcpResourcesTool, ReadMcpResourceTool].some(
    tool => tools.some(t => toolMatchesName(t, tool.name)),
  )
  if (!hasResourceTools) {
    resourceTools.push(ListMcpResourcesTool, ReadMcpResourceTool)
  }
}
中文解读

服务器声明有 resources 能力时，系统会补齐资源枚举/读取工具。

补齐过程是幂等的，不会重复注入。

同时，MCP 描述文本会做长度上限控制，防止连接器元数据吞掉上下文预算。

Skills 走 Markdown 装载，但要按来源做安全分层

`source/claude-code-source/src/skills/loadSkillsDir.ts`（第 371-373、404-467 行）

CODE 
// Security: MCP skills are remote and untrusted — never execute inline shell commands
if (loadedFrom !== 'mcp') {
  finalContent = await executeShellCommandsInPrompt(...)
}

const skillFilePath = join(skillDirPath, 'SKILL.md')
中文解读

本地 skills 可以按策略执行受控 shell 片段。

MCP 来源 skills 被视为远程不可信内容，禁用内联 shell 执行路径。

⚠️
“可读”不等于“可执行”

远程扩展内容可以参与推理，但执行权必须单独授权，不能隐式继承。

Plugins 负责更深层的运行时扩展面

`source/claude-code-source/src/types/plugin.ts` 与 `src/entrypoints/sdk/coreTypes.ts`

组件面`commands`、`agents`、`skills`、`hooks`、`output-styles`，以及可选 MCP/LSP 服务器
钩子事件`PreToolUse`、`PostToolUse`、`UserPromptSubmit`、`SessionStart`、`PostCompact` 等
治理面启停状态、安装来源、错误类型化与加载检查
1
要连外部系统

优先 MCP。

2
要复用人类工作流

优先 Skills。

3
要改运行时行为

才用 Plugins。
