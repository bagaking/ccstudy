Auto Mode 架构

Auto Mode 是分类器参与决策的自治模式。进入前要过门禁，宽泛预授权会被剥离，低风险动作走快速通道，失败路径会按交互环境分流。

先看全图：四个控制面共同决定自治边界

🚪

进入门禁

CLI 意图、settings、模型支持、fast mode breaker、组织级开关共同决定能否进入 auto。

✂️

危险规则剥离

进入 auto 前会移除宽泛 shell、PowerShell、任务相关 allow 规则，保证高风险动作仍会经过分类器。

🧮

决策流水线

先走低成本快速判定，再把高风险动作送入 transcript classifier。

↩️

退出与恢复

当 auto 失效时，系统会恢复被剥离规则，退出当前状态，并发出通知或 attachment。

🧭
阅读方式

工程文章给出安全目标；本地源码给出可执行的门禁、剥离、判定和恢复机制。

中文解读

旧的 always-allow 集合中可能存在宽泛规则，这些规则会让高风险动作绕过分类器。

系统会识别宽泛 shell、解释器和任务规则，并在进入 auto 时从当前权限上下文中移除。

被移除的规则会被暂存，退出 auto 后再恢复。

⚠️
关键原因

如果保留 `Bash(*)` 或宽泛 subagent allow 规则，最危险的动作将直接跳过分类器审查。
