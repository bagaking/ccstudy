Headless、CI 与程序化运行

`--print` 路径是一套可编排协议：输入输出格式、结构化结果、会话恢复与失败策略都由代码显式约束，适合自动化环境稳定执行。

自动化执行模型：四个可验证契约

🖨️

Headless 入口

`-p/--print` 切换到非交互执行，主流程进入 `runHeadless(...)`。

📡

NDJSON 协议

`--output-format=stream-json` 输出分帧消息，并携带 control request/response。

🧱

结构化输出

`--json-schema` 注入 `StructuredOutput` 工具，带 schema 校验与重试上限。

🔁

会话连续性

`--continue`/`--resume` 复用历史会话，支持 fork 与消息游标截断。

🧭
工程规则

CI 中优先使用可解析输出和显式停止条件（`maxTurns`、预算、schema）。避免依赖终端交互行为。

中文解读

`--json-schema` 会在 headless 模式下追加内部工具 `StructuredOutput`。

schema 使用 Ajv 编译校验，基于对象引用做缓存，降低重复脚本调用开销。

连续输出不满足 schema 时会触发上限错误并结束回合，返回类型化结果。

Tool contract 模型在最终阶段调用 `StructuredOutput`，输入必须符合 schema。

中文解读

`--continue` 读取当前目录最近会话；`--resume` 读取显式会话标识（UUID/JSONL/URL）。

`--fork-session` 决定是否复用原 session ID，便于保留原轨迹并创建分支。

`--resume-session-at` 允许按 message UUID 截断历史，得到稳定可重放的分叉点。

1
确定恢复来源

`--continue` 或 `--resume`。

2
确定会话身份策略

复用原 ID 或 `--fork-session` 新建分支 ID。

3
确定回放游标

需要时使用 `--resume-session-at` 截断历史。

选输出协议

B
设停止边界

组合 `--max-turns`、`--max-budget-usd`、`--json-schema`。

C
让错误前置

参数冲突、sandbox 不可用、无效恢复输入都在执行前失败。

D
保留观测点

headless profiler 与 stream watchdog 可定位慢启动、无事件流和卡流问题。
