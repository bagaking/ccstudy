成本与 Token 经济学

Claude Code 把成本、Token、上下文压力拆成独立账本。看懂这些账本的计算位置，才能做稳定的预算和运行策略。

一个会话会同时更新多套经济指标

同一轮响应可以同时影响美元成本、Token 分类计数、时延指标、代码改动规模。成本治理是系统问题，单看一个数字会丢失关键信号。

💵

计费账本

累计 `totalCostUSD` 和按模型拆分的 USD，再统一格式化输出。

🧮

Token 账本

输入、输出、cache read、cache creation、web search 请求分别统计。

📏

运行账本

API 时长、墙钟时长、代码增删行、工具信号按会话持久化。

⚙️
同一条响应会联动多账本

当响应里既有 token usage 又有 tool 行为时，成本、预算压力、代码行为统计会一起变化。

中文解读

有的函数把 cache token 计入总量，有的函数按预算公式排除 cache，语义目标不同。

`tokenCountWithEstimation()` 是阈值判断主函数，组合了“最近 usage + 后续消息估算”。

选错函数会直接导致 compact 或预算判断偏离真实上下文压力。

`getTokenCountFromUsage`输入+输出+缓存读写总和。
`finalContextTokensFromLastResponse`用于 task_budget 剩余额度语义，排除 cache。
`tokenCountWithEstimation`用于阈值控制，覆盖 split assistant record 场景。
