Agent 团队与通信系统

多代理协作是否稳定，取决于协调协议、消息传输、状态队列和通知回流是否一致。Claude Code 把这些能力都做成了显式运行时结构。

团队通信属于核心运行时能力

🎯

协调层

Coordinator prompt 定义委派、回收结果和验证分工。

📬

传输层

队友消息落到 mailbox 文件，使用锁和重试保证并发安全。

🧭

状态层

REPL 与 AppState 维护 worker 权限请求、网络审批、阻塞状态队列。

📡

回流层

`print.ts` 在 headless 模式继续解析通知并喂回模型回合。

💡
协作质量由协议质量决定

消息类型和路由边界不清晰时，常见结果是重复执行、错误恢复慢、上下文回收失败。
