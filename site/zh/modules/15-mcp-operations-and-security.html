MCP 运维与安全

MCP 上线后是运维问题：配置作用域优先级、OAuth 令牌生命周期、输出上限策略、信任边界和故障签名排查。

先建立运行模型：五层控制面

🧩

配置作用域

服务器清单来自多来源合并，受优先级与策略过滤约束。

🔐

认证与令牌

包含 OAuth 超时、令牌刷新、step-up 提权和 needs-auth 缓存。

📦

输出预算

超大结果会截断或落盘，避免上下文窗口被单次输出占满。

🛡️

信任边界

MCP 返回内容按外部输入处理，命令文案明确提示信任风险。

🩺

故障签名

timeout、needs-auth、session expired、连接失败需要分开定位。

📌
排障规则

先判断问题落在配置、认证、传输、输出、策略哪个层，再做动作。这样可以快速收敛。

`MCP_TIMEOUT`、`MCP_TOOL_TIMEOUT`、session 过期、认证失败是四类不同问题。

session 过期后会清理连接缓存，下一次调用会重建会话。

`claude mcp list/get` 可以快速确认当前是连接问题还是认证问题。

1
先看错误签名

区分 `needs-auth`、timeout、session expired。

2
再查配置有效集

确认目标服务器没有被作用域覆盖或策略过滤掉。

3
最后做健康复检

用 `mcp list/get` 验证修复是否生效。
