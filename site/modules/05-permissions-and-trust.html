Permissions and Trust

High capability without explicit policy boundaries is risk. Permission design is a precision control layer, not UI ceremony.

Permission Modes Are Trust Profiles

defaultAsk before sensitive operations
acceptEditsAuto-accept edit-oriented operations
bypassPermissionsBroad auto-allow mode (high-risk)
planPlanning mode with constrained execution behavior

Safe configuration means tight rule scope: narrow allowlists first, broad patterns last.

What Counts as Dangerous Bash Rule

`source/claude-code-source/src/utils/permissions/permissionSetup.ts` (lines 94-147)

CODE
export function isDangerousBashPermission(toolName: string, ruleContent: string | undefined): boolean {
  if (toolName !== BASH_TOOL_NAME) return false
  if (ruleContent === undefined || ruleContent === '') return true
  const content = ruleContent.trim().toLowerCase()
  if (content === '*') return true
  for (const pattern of DANGEROUS_BASH_PATTERNS) { ... }
  return false
}
PLAIN ENGLISH

Blank rule content means effectively “allow everything.”

Wildcard `*` is treated as dangerous by definition.

Interpreter and broad execution patterns are blocked from silent auto-allow flows.

Dangerous Pattern Sources and Circuit Breaker

  'python', 'node', 'deno', 'tsx', 'ruby', 'perl', 'php', 'lua',

  'bash', 'sh', 'ssh',

  ...CROSS_PLATFORM_CODE_EXEC, 'zsh', 'fish', 'eval', 'exec', 'env', 'xargs', 'sudo',

PLAIN ENGLISH

These patterns represent broad code execution vectors.

Auto mode strips over-broad allows that would bypass classifier safety intent.

This is layered defense, not trust-by-default.

`source/claude-code-source/src/utils/permissions/autoModeState.ts` (lines 27-33)

export function setAutoModeCircuitBroken(broken: boolean): void {
  autoModeCircuitBroken = broken
}

export function isAutoModeCircuitBroken(): boolean {
  return autoModeCircuitBroken
}
PLAIN ENGLISH

Auto mode has a circuit-breaker state to prevent unsafe silent re-entry.

Once tripped, higher-level logic can enforce explicit recovery path.

Permission Decision Path

1
Tool request arrives

Model emits structured request with tool name and arguments.

2
Rule matching runs

Exact, prefix, wildcard shapes are evaluated against policy sets.

3
Dangerous broad allows stripped

Auto mode removes unsafe blanket approvals.

4
Allow / ask / deny

Execution proceeds only after policy resolution.

⚠️
Broad permission is often implicit arbitrary code execution

`Bash(*)` convenience can silently become unrestricted command authority.
