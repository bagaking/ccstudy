Proactive, Background, and Scheduled Agents

Claude Code runs autonomous work through three execution paths: proactive ticks, backgrounded session tasks, and cron-triggered prompts. They share the same command queue, with explicit routing by agent identity and mode.

One runtime, three autonomy loops

⏱️

Proactive Loop

Injects periodic hidden `<tick>` prompts so the model can continue work without a human message every turn.

🧵

Background Session

Moves a running main query into a task with its own transcript sidechain, while the foreground UI stays responsive.

🗓️

Scheduled Task

Cron scheduler fires prompts from `.claude/scheduled_tasks.json` and enqueues them as low-priority meta commands.

🧭
Design rule

Autonomy paths are useful only when queue routing, agent scoping, and visibility controls are explicit. This module focuses on those boundaries.

Sleep/tool availability depends on proactive state, so activation happens before `getTools()`.

The runtime then keeps autonomy alive by adding hidden tick prompts only when the queue is idle.

Coordinator mode is excluded so proactive instructions do not conflict with coordinator-specific delegation logic.

  onFireTask: task => {

    else enqueuePendingNotification({ mode: 'prompt', priority: 'later', isMeta: true, workload: WORKLOAD_CRON })

// query.ts

return cmd.mode === 'task-notification' && cmd.agentId === currentAgentId
PLAIN ENGLISH

Cron fires are not all equivalent: teammate-bound tasks route to a specific agent; lead tasks route to the main queue.

Queue draining is agent-scoped, so subagents cannot consume main-thread prompts.

`isMeta: true` keeps system-generated cron prompts out of normal transcript noise.

PLAIN ENGLISH

Local cron uses `.claude/scheduled_tasks.json` and an in-process scheduler with lock semantics.

Remote scheduling is a skill-backed control flow over trigger APIs and cloud session policy gates.

Treat them as two operational surfaces: local daemon timing versus remote CCR orchestration.

Local Cron File-backed schedule, queue enqueue, REPL/SDK runtime ownership
Remote Trigger Cloud execution, environment selection, connector validation, remote policy checks
