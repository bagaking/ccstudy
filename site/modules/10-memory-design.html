Memory Design: Durable Recall Without Drift

Claude Code memory is a bounded, typed file system with strict path controls, selective retrieval, and eventual team sync. The design trades raw volume for reliability.

System shape: index, typed files, retrieval, sync

🗂️

Entrypoint index

MEMORY.md is a compact index of pointers. It is always loaded, capped, and intentionally concise.

🏷️

Typed memory files

Each memory lives in its own markdown file with frontmatter and one of four types: user, feedback, project, reference.

🔁

Scoped sync

Team memory is project-scoped, path-validated, and synced with pull-first + checksum-based delta push.

📌
Core memory design rule

Store only context that cannot be derived from current code or git state. This keeps memory high-signal and future-useful.

PLAIN ENGLISH

`MEMORY.md` is treated as a bounded index, not an unbounded notebook.

The runtime applies two independent caps: max lines and max bytes, then appends a warning when either cap fires.

This prevents long-line failure modes and protects context budget predictability.

Team memory writes use two-pass containment checks

`source/claude-code-source/src/memdir/teamMemPaths.ts`

CODE 
export async function validateTeamMemKey(relativeKey: string): Promise<string> {
  sanitizePathKey(relativeKey)
  const fullPath = join(getTeamMemPath(), relativeKey)
  const resolvedPath = resolve(fullPath)
  if (!resolvedPath.startsWith(getTeamMemPath())) throw new PathTraversalError(...)
  const realPath = await realpathDeepestExisting(resolvedPath)
  if (!(await isRealPathWithinTeamDir(realPath))) throw new PathTraversalError(...)
  return resolvedPath
}
PLAIN ENGLISH

First pass blocks obvious escapes: null bytes, encoded traversal, backslashes, absolute paths, and `..` escapes after resolve.

Second pass resolves symlinks on the deepest existing ancestor, then verifies the canonical path still sits inside team memory.

This closes symlink escape and dangling-link write vectors.

🛡️
String checks alone are insufficient

`path.resolve()` removes `..` segments, but symlink targets can still jump outside the directory. Realpath containment closes that gap.
