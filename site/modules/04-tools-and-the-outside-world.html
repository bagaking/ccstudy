MCP Session Expiry Detection

`source/claude-code-source/src/services/mcp/client.ts` (lines 193-205)

CODE
export function isMcpSessionExpiredError(error: Error): boolean {
  const httpStatus = 'code' in error ? (error as Error & { code?: number }).code : undefined
  if (httpStatus !== 404) return false
  return (
    error.message.includes('"code":-32001') ||
    error.message.includes('"code": -32001')
  )
}
PLAIN ENGLISH

A plain 404 is ambiguous.

The runtime checks JSON-RPC code `-32001` to classify session-expired state.

Correct classification enables reconnect/retry logic instead of wrong failure handling.

Context Protection in Connector Metadata

`source/claude-code-source/src/services/mcp/client.ts` (~215-220)

CODE
/**
 * Cap on MCP tool descriptions and server instructions sent to the model.
 * ... observed dumping 15-60KB ...
 */
const MAX_MCP_DESCRIPTION_LENGTH = 2048
PLAIN ENGLISH

Connector metadata can explode token usage.

Description capping keeps MCP integrations from silently consuming context budget.

This is a reliability and cost control move, not just formatting.
